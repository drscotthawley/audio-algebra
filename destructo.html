# TODO: move most of this stuff to an import 
def encode_it(demo_reals, module):
    encoder_input = demo_reals
    
    if module.pqmf_bands > 1:
        encoder_input = module.pqmf(demo_reals)
    
    encoder_input = encoder_input.to(module.device)
    demo_reals = demo_reals.to(module.device)
    noise = torch.randn([demo_reals.shape[0], 2, module.demo_samples]).to(module.device)
    
    with torch.no_grad():
        embeddings = module.encoder_ema(encoder_input)
        if module.quantized:
            #Rearrange for Memcodes
            embeddings = rearrange(embeddings, 'b d n -> b n d')
            embeddings, _= module.quantizer_ema(embeddings)
            embeddings = rearrange(embeddings, 'b n d -> b d n')
        
        embeddings = torch.tanh(embeddings)
        return embeddings, noise

# \(8.\) ‚ùî Afterword: Researching Audio-Algebra for Effects?

Can We 1-Shot-Learn Audio Effects using These Embeddings? (Spoiler: No but let‚Äôs try‚Ä¶)

Do embeddings store anything important, and can vectors in embedding-space be meaningfully used to model audio effects?

Language models showed that operations on embeddings of words (for well-trained masked language models) could produce ‚Äúsemantically meaningful‚Äù results, such as ‚Äúking - man + woman = queen‚Äù or ‚Äújapan - tokyo = germany - berlin‚Äù.

Can our model(s) do similar things for audio? Here we‚Äôll subtract two (new) embeddings and add them to a third (your original input embedding). What do we get?

In what follows, you‚Äôll upload two ‚Äúversions‚Äù of the same audio, with (‚Äúwet‚Äù) and without (‚Äúdry‚Äù) an effect applied (e.g. distortion).

Will we get e.g. ‚Äúwet_piano - dry_piano + dry_guitar = wet_guitar‚Äù?

In the examples provided in examples/, we used an overdrive effect for ‚Äúwet‚Äù. This is a nonlinear effect, so we might not expect a ‚Äúlinear addition rule‚Äù to work for this (spoiler: and we‚Äôd be correct), but let‚Äôs try it anyway:

# @markdown Time-average the difference in embeddings?

if time_avg: 
    diff = diff.mean(axis=-1)
else:
    # make sure embeddings and diff have the same length
    length_difference = z.shape[-1] - diff.shape[-1]
    if length_difference > 0:  # zero-pad the end of diff
        diff = torch.nn.functional.pad(diff,(length_difference,0,0,0,0,0), mode='constant')
